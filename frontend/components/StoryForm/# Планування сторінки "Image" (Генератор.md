# Планування сторінки "Image" (Генератор футажів)

Цей план розбиває створення комплексної сторінки для генерації зображень на кілька логічних етапів. Оскільки задача вимагає взаємодії з API, складного управління станом на клієнті (черга, IndexedDB) та спеціального UX (Framer Motion, JSZip), реалізовувати це за один крок ризиковано і неефективно. Поетапний підхід забезпечить стабільність і дозволить тестувати кожну частину окремо.

## Етап 1: Підготовка та встановлення залежностей

**Мета:** Налаштувати необхідні бібліотеки для роботи з анімаціями, архівами та локальною базою даних у браузері.

- Встановити бібліотеки в `frontend`: `framer-motion`, `jszip`, `file-saver`, `idb-keyval` (легка обгортка над IndexedDB для простоти роботи з промісами).
- Додати типи для TypeScript (якщо потрібно): `@types/file-saver`.

## Етап 2: Створення API роутів (Next.js або NestJS)

**Мета:** Створити безпечні ендпоінти для звернень до моделей (щоб не "світити" API ключі на клієнті).

- Створити API роут `POST /api/generate-prompts` (або в NestJS) для виклику `gemini-2.5-flash` з системним промптом: "Розбий цю ідею на N детальних англомовних промптів...". Повернення масиву JSON.
- Створити API роут `POST /api/generate-image` (або в NestJS) для виклику `imagen-4.0-fast-generate-001` (або `gemini-2.5-flash-image`), що отримуватиме промпт і повертатиме зображення (Base64).
- Реалізувати можливість легкого перемикання моделей (наприклад, через змінні середовища або константи).

## Етап 3: Логіка роботи з IndexedDB та чергою генерації

**Мета:** Реалізувати клієнтські сервіси для збереження даних та послідовного виклику API.

- Написати сервіс `imageStorage.ts` для запису/читання/очищення картинок і промптів з IndexedDB (використовуючи `idb-keyval`).
- Реалізувати хук `useImageGenerator` або стейт-менеджер для:
  - Формування черги запитів (генерувати зображення по черзі, а не всі 30 одночасно).
  - Відстеження статусів кожної картки (Waiting, Generating..., Done, Error).
  - Відновлення стану з IndexedDB при оновленні сторінки.
  - Очищення IndexedDB при новій генерації.

## Етап 4: Інтерфейс користувача (UI) - Основна сторінка

**Мета:** Створити сторінку `/image` з формою та прогрес-баром.

- Створити компонент форми (ввід ідеї, кількість картинок до 30, кнопка).
- Додати загальний прогрес-бар (наприклад: "12 / 30 футажів готово").
- Обробка події `beforeunload` (попередження "Процес генерації ще триває. Ви впевнені, що хочете піти?").

## Етап 5: Інтерфейс користувача (UI) - Картки та Модальне вікно

**Мета:** Візуалізація результатів з анімаціями Framer Motion.

- Створити компонент картки-заглушки (skeleton) зі статусами і відповідними стилями.
- Додати анімації появи карток (Framer Motion).
- Створити модальне вікно для перегляду згенерованого зображення та його промпту.
- Реалізувати індикатор "Відновлено зі сховища".

## Етап 6: Завантаження файлів (Фіналізація)

**Мета:** Додати функціонал збереження результатів.

- Додати можливість скачування окремої картинки.
- Реалізувати логіку пакування всіх успішно згенерованих картинок (зі статусів Done) у ZIP архів за допомогою `jszip` та `file-saver`.
- Кнопка "Скачати все (ZIP)", що з'являється після завершення всієї черги.

## Етап 7: Тестування та обробка помилок

**Мета:** Перевірка "Safety Settings" та помилок мережі.

- Додати кнопку "Перегенерувати" на картках з помилками.
- Обробка відмов через цензуру (Safety Settings) і виведення причини користувачу.
